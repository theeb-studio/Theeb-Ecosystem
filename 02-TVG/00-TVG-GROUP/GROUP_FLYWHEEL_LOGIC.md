## THE TVG FLYWHEEL

TVG operates as a **four-layer financial stack** where data flows upward from revenue operations to intelligence and contracting, while contracts and financial products flow downward. Each layer makes the next more powerful — and the stack becomes the financial operating system of Saudi healthcare.

```
┌─────────────────────────────────────────────────────────────┐
│  LAYER 4: OUTCOME-BASED CONTRACTING (VBH Co)                │
│  Bundled payments · Shared savings · VBP contracts           │
│  ← Contracts flow DOWN to transform payment models           │
├─────────────────────────────────────────────────────────────┤
│  LAYER 3: VALUE INTELLIGENCE (VBH Co)                        │
│  PROMs · Cost-of-care analytics · AR-DRG intelligence        │
│  ← Intelligence enables contract design                      │
├─────────────────────────────────────────────────────────────┤
│  LAYER 2: RISK INFRASTRUCTURE (Nexus MGA)                    │
│  Insurance product design · Malpractice + CME bundle         │
│  ← Risk products align payer-provider incentives             │
├─────────────────────────────────────────────────────────────┤
│  LAYER 1: REVENUE INFRASTRUCTURE (Integra RCM)              │
│  AI coding · Claims management · AR-DRG readiness            │
│  ← Revenue recovery is the entry point                       │
└─────────────────────────────────────────────────────────────┘
          ↑ DATA FLOWS UP        CONTRACTS FLOW DOWN ↓
```

## HOW THE 3 ENTITIES REINFORCE EACH OTHER

### The Stack Deployment Sequence

The TVG flywheel works through **sequential deployment** at each hospital client:

```
MONTH 0:  Integra RCM enters hospital
          → Revenue recovery proves immediate ROI
          → Claims data starts flowing into TVG intelligence layer
          → Hospital trusts TVG with financial operations

MONTH 6:  VBH Co deploys PROMs collection
          → Outcomes measurement begins
          → Hospital sees quality data for first time
          → Cost-of-care analytics using Integra RCM claims data

MONTH 12: VBH Co designs VBP contracts
          → First bundled payment or shared savings contract
          → Hospital transitions from volume to value
          → Outcomes data validates the financial model

MONTH 15: Nexus MGA provides risk product
          → Data-driven insurance product offered to hospital's insurers
          → Risk-sharing contract designed using real outcomes data
          → Complete financial stack deployed

MONTH 18+: Hospital embedded across all 4 layers
           → Switching costs compound
           → Data advantage grows
           → TVG = hospital's financial operating system
```

**Compound effect:** Each client embedded in Layer 1 is a pre-qualified pipeline for Layers 2-4. CAC for higher layers approaches zero because the relationship already exists.

### The Data Reinforcement Loop

```
Integra RCM generates claims data
    → VBH Co uses claims data for cost-of-care analytics
    → VBH Co measures outcomes (PROMs)
    → Outcomes + cost data feed Nexus MGA underwriting models
    → MGA designs better insurance products
    → Better products reduce insurer loss ratios
    → Reduced losses attract more payer-provider contracts
    → More contracts generate more claims data
    → [LOOP ACCELERATES]
```

### The Insurer Crisis Resolution Loop

```
Insurer loss ratios at 83-96%
    → Payers demand VBP solutions → VBH Co provides
    → Providers need revenue optimization → Integra RCM provides
    → Risk alignment needed → Nexus MGA provides
    → VBP contracts proven → more payers demand
    → [LOOP ACCELERATES as market transforms]
```

## WHY TVG IS UNREPLICABLE

A competitor can build a billing company (Layer 1). But TVG is the only entity in Saudi Arabia building all four layers simultaneously — and the layers compound. Layer 1 data feeds Layer 3 analytics, which enables Layer 4 contracts, which generate data for Layer 2 risk models. Remove any layer and the stack loses its power.
