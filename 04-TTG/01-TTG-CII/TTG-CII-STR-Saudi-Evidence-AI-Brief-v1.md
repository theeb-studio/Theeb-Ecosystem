### How OpenEvidence Works

```
Physician asks clinical question
         ↓
RAG Engine searches 35M+ licensed papers
         ↓
Vector embeddings find most relevant sources
         ↓
LLM synthesizes answer with full citations
         ↓
"Deep Consult" mode: multi-minute comprehensive analysis
         ↓
Response grounded in SNOMED CT + MeSH ontologies
```

## 4. THE THREE-PHASE BUILD

### Phase 1: MVP on Owned + Open Content (Q2-Q4 2026)

**Goal:** Working clinical AI with Saudi context, Arabic support, demonstrable value

| Component | Source | Status |
|-----------|--------|--------|
| RAG engine | Build (standard architecture — well-documented) | Development |
| Medical ontologies | SNOMED CT, MeSH, ICD-10 (open) + SFDA | Available |
| Content corpus | Tier 1 (Waraqa, CRO) + Tier 2 (PubMed, SFDA) | Available |
| Arabic NLP | Multilingual LLM (Claude, GPT-4, open-source Arabic models) | Available |
| User interface | Web + API + mobile | Development |
| Pilot clients | 2-3 hospitals (THDG network + Care Hospitals) | In discussion |

**Deliverable:** "Ask a clinical question in Arabic or English → get evidence-based answer with Saudi context and citations in 30 seconds"

### Phase 2: Government Licensing Expansion (2027)

**Goal:** Add Tier 3 content through government-negotiated AI rights

**Approach to MOE/SDL:**
> "The Saudi Digital Library spends SAR 80M+ every 3 years on reading access to foreign databases. For an additional 15-20% (SAR 12-20M), we can negotiate AI processing rights on the same content. Result: Saudi Arabia gets a sovereign clinical AI platform — instead of depending on American platforms like OpenEvidence that license the same content and sell it back to Saudi physicians."

**Approach to publishers (Wiley model):**
> Licensed XML access for RAG/inference — NOT for model training. Multi-year agreements. Content provided with metadata tags for structured retrieval.

### Phase 3: Scale + Monetize (2027+)

**Goal:** B2B subscription platform for all Saudi healthcare institutions

| Channel | Model |
|---------|-------|
| Direct sales | CII sales team → hospitals, universities |
| Via iDev | Add CII subscription to their existing institutional sales |
| Via SDL | Bundle with existing database subscriptions |
| Via THDG-Welunion | Built into Welunion clinic technology stack |
| Via TVG-Integra RCM | Clinical coding AI assistant (feeds RCM accuracy) |
